林火检测系统 — 前端对接须知（本期）
更新时间：2026-04-18
维护：仅在本目录 `docs/前后端对接/` 更新；细节以同目录 `接口汇总.txt`、`后端自检.txt` 为准。

========================
一、本期请你（前端）做的事
========================

1) 高德 JS 地图 Key 不要写死在仓库里
   - 用户登录后请求：GET /api/geo/map-config（需 Bearer）
   - 使用返回的 data：
     - jsapi_key → 传给 AMapLoader.load({ key })
     - security_js_code → 在 load 前设置 window._AMapSecurityConfig.securityJsCode
   - 若 code=50002 且提示未配置 amap_jsapi_key：让运维在服务端 config.yaml 填写「Web端(JS API)」Key（与 Web 服务 Key 是两类，见运维说明）。

2) 区县展示 / 「未解析区县」
   - 一律走：GET /api/geo/reverse-geocode?longitude=&latitude=（GCJ-02，与地图一致）
   - 成功时 data 会展平多个字段（district、region、formatted_address 等），任取其一非空即可展示；不要只认某一个字段名。
   - 若 code=50002：读 message（可能含高德 infocode，如 10009 表示 Key 类型错误；10021/CUQPS 为配额或 QPS 超限；亦可能为服务端限流文案），给用户可读提示，不要静默当「未解析」。

3) 地名搜索
   - 继续走：GET /api/geo/place-search?q=（需 Bearer），不要浏览器直连高德 REST。

4) 火点 PATCH（可选）
   - PATCH /api/fire-markers/{id} 支持同时传 longitude、latitude（须成对），用于改点后由服务端逆地理刷新区县；若产品有「拖动标记更新坐标」应对接此字段。

5) 台账列表
   - GET /api/fire-ledger 每条 item 含 marker_id、longitude、latitude；可与列表去重/补显逻辑对齐（详见同目录后端自检 B/C）。

6) 创建火点后台账
   - 后端创建火点成功后会自动写一条 fire_marker_events，台账里应能较快看到新记录（仍可与本地临时行合并展示，按产品定）。

========================
二、不必再做的事（避免重复劳动）
========================

- 不要在业务代码里配置 amap_web_service_key（Web 服务 Key）或 amap_jsapi_key（JS Key）；均在服务端 config.yaml。
- 逆地理、POI 检索：不要在前端直连 restapi.amap.com。

========================
三、联调自测（可选）
========================

见同目录 `后端自检.txt` 中 A 节 curl 示例；替换 TOKEN 与坐标即可。
