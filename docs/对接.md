# 后端API对接文档

## 1. 基本信息

### 1.1 项目信息
- **项目名称**: ForestFire Backend
- **版本**: 1.0.0
- **API基础路径**: `http://localhost:8000/api`
- **认证方式**: JWT Token（在请求头中添加 `Authorization: Bearer <token>`）
- **请求格式**: JSON
- **响应格式**: JSON

## 2. 接口列表

### 2.1 火焰检测接口

#### 2.1.1 分析图片/视频
- **路径**: `/analyze`
- **方法**: POST
- **请求体**: 表单数据，键名为 `file`，值为图片或视频文件
- **支持的文件类型**: jpg, jpeg, png, gif, mp4, avi, mov
- **响应**:
  - 成功:
    ```json
    {
      "code": 20000,
      "message": "成功",
      "data": {
        "image_info": {
          "width": 640,
          "height": 480,
          "channels": 3
        },
        "detections": [
          {
            "class": "fire",
            "confidence": 0.95,
            "bbox": [100, 100, 200, 200]
          }
        ],
        "fire_probability": 0.95,
        "risk_level": "high",
        "output_image_path": "uploads/image_output.jpg",
        "fire_count": 2,
        "average_confidence": 0.9,
        "file_url": "/uploads/image_output.jpg",
        "file_base64": "base64编码的文件内容",
        "file_type": "image/jpeg"
      }
    }
    ```
  - 失败:
    ```json
    {
      "code": 40000,
      "message": "不支持的文件类型，支持的类型: jpg, jpeg, png, gif, mp4, avi, mov",
      "data": null
    }
    ```

### 2.1.2 注意事项
- 上传的文件会被保存在 `uploads/` 目录中
- 分析后的图片/视频会以 `原始文件名_output.扩展名` 的格式保存在同一目录
- 响应中包含火焰数量、平均置信度、风险等级评估、文件URL和文件的base64编码
- 风险等级评估规则：火焰数量 > 5 为高风险，3-5 为中风险，≤ 3 为低风险
- `file_base64` 字段包含处理后文件的base64编码，可以直接在前端显示
- `file_type` 字段包含文件的MIME类型，用于正确处理base64编码的文件

## 3. 错误码说明

| 错误码 | 含义 | HTTP状态码 |
|-------|------|-----------|
| 20000 | 成功 | 200 |
| 40000 | 请求参数错误 | 400 |
| 40100 | 未授权 | 401 |
| 40300 | 禁止访问 | 403 |
| 40400 | 资源不存在 | 404 |
| 50000 | 服务器内部错误 | 500 |
| 50001 | 数据库错误 | 500 |
| 50002 | 第三方服务错误 | 500 |

## 4. 数据格式规范

### 4.1 标准响应结构
```json
{
  "code": 20000,
  "message": "成功",
  "data": { /* 响应数据 */ }
}
```

### 4.2 错误响应结构
```json
{
  "code": 40000,
  "message": "错误信息",
  "data": null
}
```

## 5. 对接建议

### 5.1 前端对接
- 使用统一的API请求封装
- 处理各种错误码
- 实现请求超时和重试机制
- 对敏感操作进行二次确认
- 缓存常用数据，减少API调用

### 5.2 常见问题
- **token过期**: 重新登录获取新token
- **网络错误**: 实现离线模式或重试机制
- **权限不足**: 引导用户授权或切换账号
