# 后端运行环境配置

## 1. 环境要求

### 1.1 Python 版本

- **Python 3.9+**（必需：用户/火点模块使用 SQLAlchemy 2.x 与当前类型注解写法）

### 1.2 数据库

- **MySQL 8.x**（或兼容的 MariaDB）：用于用户表、火点标记表持久化  
- 建库：在 MySQL 中执行仓库内 `scripts/init_mysql.sql`（创建 `forestfire` 库；**表由应用首次启动时自动创建**）

### 1.3 操作系统

- Windows / Linux / macOS 均可

---

## 2. 依赖包

以项目根目录 **`requirements.txt`** 为准；安装：

```bash
pip install -r requirements.txt
```

### 2.1 依赖列表（与 requirements.txt 对应）

```
fastapi
uvicorn
pyyaml
python-multipart
Pillow
numpy
ultralytics
opencv-python
sqlalchemy>=2.0
pymysql>=1.1.0
python-jose[cryptography]
passlib[bcrypt]
bcrypt<4
```

说明：`passlib` 1.7.x 会读取 `bcrypt.__about__` 获取版本信息，而 **bcrypt 4.x** 已移除该属性，可能在日志中出现 `(trapped) error reading bcrypt version`。仓库 `requirements.txt` 已固定为 **`bcrypt<4`**，与上表一致。

### 2.2 依赖说明

| 包 | 用途 |
|----|------|
| **fastapi** | Web 框架与 OpenAPI |
| **uvicorn** | ASGI 服务，运行应用 |
| **pyyaml** | 读取 `config/config.yaml` |
| **python-multipart** | 文件上传（`/api/analyze`） |
| **Pillow** / **numpy** | 图像与数值（检测链路常用） |
| **ultralytics** | YOLO 火焰检测 |
| **opencv-python** | 图像/视频帧处理 |
| **sqlalchemy** | ORM，连接 MySQL |
| **pymysql** | MySQL 驱动（连接串形如 `mysql+pymysql://...`） |
| **python-jose** | JWT 签发与校验（算法见配置 `jwt_algorithm`，默认 HS256） |
| **passlib** + **bcrypt** | 用户密码**哈希**（bcrypt，非可逆加密） |

### 2.3 验证安装（可选）

```bash
pip list
```

或抽查：

```bash
pip show fastapi sqlalchemy pymysql pyyaml passlib bcrypt
```

Windows PowerShell 无 `grep` 时，可直接在 `pip list` 输出中搜索包名。

## 3. 安装步骤

### 3.1 安装 Python

从 [Python 官网](https://www.python.org/downloads/) 安装 **3.9 或更高** 版本。

### 3.2 创建数据库

1. 启动 MySQL  
2. 执行 `scripts/init_mysql.sql` 创建库 `forestfire`（字符集 `utf8mb4`）

### 3.3 编辑配置文件

编辑 **`config/config.yaml`**（与 `config/config.py` 同目录）：

- **`database_url`**：与本地 MySQL 账号、库名一致  
- **`jwt_secret`**：生产环境务必改为**足够长且随机**的密钥  
- **`environment`**：`development` 或 `production`

**说明**：项目**不使用环境变量**，可调参数均在 `config.yaml`。

### 3.4 安装 Python 依赖

在项目根目录（含 `main.py`、`requirements.txt` 的目录）执行：

```bash
pip install -r requirements.txt
```

### 3.5 模型文件

将 **`yolo10n_fire.pt`** 放在**项目根目录**（与 `main.py` 同级）。检测服务在启动时会加载该文件。

## 4. 运行与验证

### 4.1 启动服务

```bash
python main.py
```

默认监听 `http://0.0.0.0:8000`（见 `main.py`）。

### 4.2 验证

| 地址 | 说明 |
|------|------|
| `http://localhost:8000/` | 根路径，返回项目信息 |
| `http://localhost:8000/docs` | Swagger 交互文档 |
| `http://localhost:8000/api/...` | 业务 API（前缀见 `config.yaml` 的 `api_v1_str`） |
| `http://localhost:8000/uploads/...` | 分析结果等静态文件（由应用挂载上传目录） |

### 4.3 生产环境启动示例

```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```

建议前置 **Nginx** 等反向代理、**HTTPS**，并在 `config.yaml` 中将 `environment` 设为 `production`，同时收紧 CORS（当前开发配置见 `main.py`）。

## 7. 注意事项

- **`config/config.yaml` 必须存在**，否则启动会因找不到配置而失败。  
- **`uploads/`**：应用启动时会尝试创建（见 `main.py`）；需保证进程对该路径有写权限。  
- **视频分析**耗时与机器性能相关，大文件建议在前端或网关侧限制大小与时长。  
- **安全**：勿将含真实生产密码、`jwt_secret` 的 `config.yaml` 提交到公开仓库；CORS 生产环境应改为明确的前端域名。

## 8. 故障排除

### 8.1 模型加载失败

- 确认项目根目录存在 `yolo10n_fire.pt`  
- 升级或固定 `ultralytics` 版本，避免与权重格式不兼容

### 8.2 数据库连接失败

- 核对 `config.yaml` 中 `database_url` 的主机、端口、库名、用户名与密码  
- 确认已执行 `init_mysql.sql` 且 MySQL 服务已启动  
- 防火墙或云安全组是否放行 MySQL 端口

### 8.3 缺少配置文件

- 报错提示缺少 `config/config.yaml` 时，在 `config/` 下创建该文件（可参考仓库中已有示例结构）

### 8.4 依赖安装失败

- 执行 `python -m pip install --upgrade pip` 后重试  
- Windows 下部分包编译失败时，可安装 **Visual C++ Build Tools**

### 8.5 端口占用

- 修改 `main.py` 中 `uvicorn.run` 的 `port`，或结束占用 8000 端口的进程

### 8.6 SQLAlchemy / typing 相关导入错误

- 确认已安装 **SQLAlchemy 2.x**（`pip show sqlalchemy`）  
- 确认 Python 版本 **≥ 3.9**
